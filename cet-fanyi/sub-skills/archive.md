# 知识归档子技能（archive）

## 功能

从会话上下文中识别可积累的知识，归档到 KBase 目录。

## KBase 目录结构

```
KBase/
├── technique.md    ← 翻译技巧速查（最多 10 条，总字符 ≤ 1000）
├── 001/            ← 第一个归档单元
├── 002/            ← 第二个归档单元
└── ...
```

## 工作流

### Step 1：扫描会话上下文

回顾当前会话，识别以下类型的可积累内容：
- **词汇搭配**：高频固定搭配、易错词组
- **句型结构**：常用句式模板、连接词用法
- **翻译技巧**：从练习中总结出的实战经验
- **易错点**：翻译过程中踩过的坑

### Step 2：归档到 KBase

#### 技巧类 → technique.md

精炼为一句简短的技巧条目，通过脚本写入：

```bash
# 先检查是否能添加
python3 skills/cet-fanyi/scripts/kbase.py check "技巧内容"

# 确认可添加后写入
python3 skills/cet-fanyi/scripts/kbase.py add "技巧内容"
```

脚本会自动校验条目数（≤10）和总字符数（≤1000），超限时返回 error，需要先用 remove 删除旧条目腾出空间。

```bash
# 查看当前所有技巧
python3 skills/cet-fanyi/scripts/kbase.py list

# 删除指定编号的技巧
python3 skills/cet-fanyi/scripts/kbase.py remove <编号>
```

**权重择优机制**：当 technique.md 已满 10 条且需要插入新技巧时，按以下权重评估，保留权重最高的 10 条：

| 权重 | 标准 | 示例 |
|------|------|------|
| 10 | 核心应试策略，适用于所有话题 | 上位词替代、释义法、保大词跳小词 |
| 8 | 高频考点，几乎每次都会遇到 | 比喻不直译、并列形式一致 |
| 7 | 常见易错陷阱，中英直译差异大 | "基本"≠basically、"紧张"≠nervous |
| 6 | 特定搭配/句型，提升表达准确性 | avoid+V-ing、"不予"用without |
| 4 | 特定场景表达，适用面较窄 | 条款文体用语、特定句式 |

**操作流程**：
1. 对新技巧和现有 10 条分别评估权重
2. 若新技巧权重 > 现有最低权重条目，用 `remove` 删除最低权重条目，再 `add` 新技巧
3. 若新技巧权重 ≤ 现有最低权重，跳过，不替换
4. 同权重的，优先保留已在列表中的（稳定性原则）

#### 知识点类 → 编号目录

词汇、句型、易错点等较长的知识点，写入 KBase 下按序号递增的目录：

1. 先查看已有目录，确定下一个编号（如已有 001、002，则新建 003）
2. 在新编号目录下创建文件，按内容分类命名

```
KBase/001/
├── vocab.md        ← 词汇积累
├── patterns.md     ← 句型积累
└── mistakes.md     ← 易错点
```

### Step 3：确认归档结果

向用户展示本次归档的内容摘要，包括：
- 新增了哪些技巧到 technique.md
- 新建了哪些编号目录及文件
- 当前 technique.md 的剩余容量

## 规则

1. technique.md 每条技巧只能写一个要点
2. technique.md 超限时必须先提示用户选择删除哪些旧条目，不能自行覆盖
3. 编号目录只增不改，不删除历史归档
